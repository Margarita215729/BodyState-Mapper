1 · Data Intake

Not a diagnostic system — traceable hypotheses only.

Enter what is known. Leave unknown fields as unknown / unavailable — unknown is not negative evidence. Inputs map to nodes in the integrated evidence graph; the engine is a transparent, rule-based mapping (no machine learning), and outputs only traceable hypotheses with explicit uncertainty.

2 · Traceable Report

This is not a diagnosis. Outputs are traceable hidden-state hypotheses with uncertainty, missing data, guardrails, and next-best measurements.

Fill in observations above and select Generate report. Every output below will be a traceable hypothesis with explicit uncertainty — never a diagnosis.

3 · Activated Evidence Graph

The activated subgraph shows the full traceable chain from input signals to candidate hidden states, the guardrails that constrain them, and the measurements that would reduce uncertainty. Click any node or edge for full detail.

Generate a report to activate the relevant subgraph of the 1,979-node evidence graph.

Select a node or edge in the activated graph to inspect its full traceable detail.

4 · Next Best Measurements

Ranked measurements that would most reduce uncertainty for the active hypotheses. These are recommendations to resolve ambiguity — not instructions for treatment.

Next-best measurements appear here after a report is generated.

5 · Source Traceability

Every activated node and edge traces back to source files and source trees in the corpus. The full corpus-wide view is in the Source Traceability tab.

The sources behind the activated hypotheses appear here after a report is generated.
